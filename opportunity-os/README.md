# Opportunity OS

**Start where you are. Go where you want.**

Opportunity OS is an AI-guided planning workspace for adults who may be starting without a high school diploma, traditional credentials, strong technology confidence, or a clear career path. It turns a learner’s goal, current situation, strengths, and available time into a respectful, realistic roadmap with small next steps.

## OpenAI Build Week submission

### Problem
Career and education systems often assume that people already know the language, tools, credentials, and next steps needed to move forward. That creates extra friction for learners who are navigating technology, education, and employment at the same time.

### Solution
Opportunity OS uses the OpenAI Responses API and Structured Outputs to generate:

- a personalized north star and momentum message;
- three to five practical actions sized to the learner’s available time;
- opportunity categories to investigate and verify;
- a plain-language AI guide for follow-up questions;
- built-in digital, career, and money-skills prototype lessons.

### Responsible design

- The OpenAI API key remains server-side.
- API responses use strict JSON schemas before rendering.
- The app does not claim that a job, program, deadline, salary, or funding source is current.
- Users are reminded to verify details with official sources.
- Prompts discourage sensitive personal information and shame-based language.
- OpenAI response storage is disabled with `store: false`.

## Demo modes

Opportunity OS is easy to judge and demonstrate in two modes:

1. **OpenAI mode:** set `OPENAI_API_KEY`; roadmaps and guide responses are generated by the configured model.
2. **Demo mode:** omit the API key or set `DEMO_MODE=true`; the full interface works with deterministic sample responses and no API charges.

The `/api/health` endpoint reports whether the running app is in `openai` or `demo` mode.

## Run locally

Requirements: Node.js 20 or newer.

```bash
npm install
cp .env.example .env
npm run dev
```

On Windows PowerShell:

```powershell
npm install
Copy-Item .env.example .env
npm run dev
```

Open `http://localhost:3000`.

To use OpenAI mode, add your key to `.env`:

```env
OPENAI_API_KEY=your_key_here
OPENAI_MODEL=gpt-5-mini
DEMO_MODE=false
PORT=3000
```

Never commit `.env` or place the key in files under `public/`.

## Validate before a demo

```bash
npm run check
npm start
```

Then verify:

1. `/api/health` returns `status: "ok"`.
2. “Build my roadmap” produces a complete roadmap.
3. The AI guide returns an answer and one small next step.
4. Skills Studio lessons open and clearly identify prototype-only actions.
5. Opportunity Finder labels its results as categories that require verification.
6. Social profile links reject non-HTTPS and lookalike domains.
7. Browser source and network payloads never contain the API key.

## Deploy to Render

A `render.yaml` Blueprint is included.

1. Create a new Render Blueprint from this repository.
2. Add `OPENAI_API_KEY` as a secret environment variable for OpenAI mode.
3. Keep `DEMO_MODE=false`; without a key, the app safely falls back to demo mode.
4. Deploy and test `/api/health` before sharing the public URL.

The app also works on Railway, Fly.io, or any Node host that provides a `PORT` environment variable.

## Architecture

```text
Browser (HTML/CSS/JS)
        |
        | JSON only
        v
Express server
  |-- /api/roadmap  -> OpenAI Responses API + strict schema
  |-- /api/guide    -> OpenAI Responses API + strict schema
  |-- /api/profile  -> session-only prototype profile
  |-- /api/health   -> deployment health and mode
        |
        v
OpenAI model (server-side key only)
```

## Current prototype limitations

- Opportunity Finder shows representative categories, not live listings.
- Profile links are stored only in server memory and reset when the service restarts.
- Skills Studio contains prototype lesson cards rather than a complete learning management system.
- The project does not provide legal, financial, admissions, or employment guarantees.

## Suggested 90-second demo

1. Explain the access problem and target user.
2. Enter a goal, current situation, strengths, and weekly time.
3. Generate the roadmap and point out the time-sized actions.
4. Open one suggested path in Opportunity Finder.
5. Ask the AI guide a follow-up question.
6. Close with: “Opportunity OS does not decide a person’s future. It helps them see and take the next possible step.”
